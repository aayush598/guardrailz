# Errors

Guardrails APIs return **structured, predictable errors** to simplify handling and debugging.



## Error Format

```json
{
  "error": "InvalidProfile",
  "message": "Profile 'unknown-profile' does not exist"
}
````



## Common Errors

### Unauthorized

```json
{
  "error": "Unauthorized",
  "message": "Authentication required"
}
```

### Invalid Profile

```json
{
  "error": "InvalidProfile",
  "message": "Profile not found"
}
```

### Validation Failed

```json
{
  "error": "ValidationFailed",
  "message": "One or more guardrails failed"
}
```



## HTTP Status Codes

| Code | Meaning            |
| - |  |
| 400  | Invalid request    |
| 401  | Unauthorized       |
| 403  | Forbidden          |
| 404  | Resource not found |
| 429  | Rate limited       |
| 500  | Internal error     |



## SDK Error Handling

The SDK throws typed errors:

```ts
try {
  await client.validate(...)
} catch (err) {
  console.error(err.message);
}
```



## Best Practices

* Log error `code` and `message`
* Retry only idempotent requests
* Treat `429` as backpressure
