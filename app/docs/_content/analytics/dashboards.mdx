# Analytics Dashboards

Dashboards turn analytics data into **clear, actionable visibility**.

They provide real-time and historical views of guardrail behavior.



## Dashboard goals

Guardrails dashboardsx are designed to:

- Surface safety issues quickly
- Highlight abnormal behavior
- Track long-term trends
- Support audits and reviews

They are optimized for both engineers and non-technical stakeholders.



## Core dashboard views

### Overview dashboard

Shows:

- Total executions
- Pass / fail ratio
- Average execution time
- Success rate trends

This is the primary health view.



### Guardrail performance dashboard

Displays:

- Most executed guardrails
- Highest failure rates
- Slowest guardrails

Used to tune and optimize configurations.



### Profile analytics dashboard

Tracks:

- Usage per profile
- Success rates by profile
- Profile-level risk exposure

Helpful for comparing safety strategies.



### Usage & rate limits

Visualizes:

- Requests over time
- Rate limit consumption
- API key usage

Critical for operational monitoring.



## Operational value

Dashboards help teams:

- Detect attacks early
- Prevent regressions
- Prove compliance
- Optimize cost and latency
